Overview

The class for all visual sprites in the game. Sprite images are determined by their coordinates on the SpriteSheet. This is best used as an array, however constants and Hash keys can also be used to make development easier. These traits are expressed with the #name and #number attributes. A Spriteâ€™s name is the symbol used to look it up in the SPRITE_HASH. Its number are the coordinates those names access. Sprites are not meant to exist outside of a master object. This makes creating and destroying them both easier: instead of tracking them separately, in tandem, only the master object needs to be managed. The sprite image is changed through the #change_name! method.)
